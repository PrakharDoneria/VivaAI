# Security Policies and Procedures

This document outlines security procedures and general policies for the `VivaAI` project.

* [Reporting a Bug](#reporting-a-bug)
* [Disclosure Policy](#disclosure-policy)
* [Comments on this Policy](#comments-on-this-policy)

## Reporting a Bug

The `VivaAI` team and community take all security bugs in `VivaAI` seriously.
Thank you for improving the security of `VivaAI`. We appreciate your efforts
and responsible disclosure and will make every effort to acknowledge your contributions.

Report security bugs by opening a private issue or contacting the lead maintainer
@PrakharDoneria directly on GitHub.

The lead maintainer will acknowledge your report within 48 hours, and will send
a more detailed response indicating the next steps in handling your report.

## Disclosure Policy

When the security team receives a security bug report, they will assign it to a
primary handler involving the following steps:

* Confirm the problem and determine the affected versions.
* Audit code to find any potential similar problems.
* Prepare fixes for all releases still under maintenance.

## Comments on this Policy

If you have suggestions on how this process could be improved please submit a pull request.
